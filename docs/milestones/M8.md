# M8 - Refresh System

## Status

Completed on 2026-04-24.

## Implemented

- Added lightweight FastAPI background scheduler without extra heavy dependencies.
- Regular refresh interval defaults to 30 minutes.
- Pre-lock refresh interval defaults to 5 minutes when a game starts within the configured window.
- Added a process-local async lock so refresh jobs do not run in parallel.
- Added full refresh cycle:
  - source refreshes
  - aggregation refresh
  - projection refresh
- Added manual refresh endpoint: `POST /refresh/run`.
- Scheduler can be disabled through `SCHEDULER_ENABLED`.

## Resource Notes

The scheduler uses the same lightweight source adapters and shared source-health handling as manual refreshes. It does not start browser instances and will skip overlapping refresh cycles.
