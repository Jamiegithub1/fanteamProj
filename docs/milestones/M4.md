# M4 - Playzilla Integration

## Status

Completed on 2026-04-24.

## Implemented

- Modular source adapter layer for bookmaker integrations.
- Playzilla as the mandatory first source.
- Lightweight discovery for the current Playzilla redirect, SPA bundles, and Altenar WSDK assets.
- Parser support for NBA player props: Points, 3PT Made, Rebounds, Assists, Steals, Blocks, Turnovers, Double-Double, Triple-Double.
- Database persistence for bookmaker, markets, raw odds, refresh runs, and source health.
- API endpoints:
  - `GET /sources/health`
  - `POST /sources/playzilla/refresh`

## Resource Notes

Playzilla currently embeds sportsbook data through the Altenar WSDK. The public lightweight endpoints require a WSDK validation token, so the adapter records a degraded source-health state when unauthenticated direct HTTP access is blocked. This keeps the local app stable without running a heavy browser scraper or overloading the server.
