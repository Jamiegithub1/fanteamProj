# M10 - Auth & Security

## Status

Completed on 2026-04-24.

## Implemented

- Added HTTP Basic auth for protected backend endpoints.
- Credentials are configured through ENV:
  - `APP_USERNAME`
  - `APP_PASSWORD`
- Health endpoints remain public for Docker/local checks.
- Frontend now shows a login screen and sends the Basic auth header for app data and refresh calls.
- Added logout that clears the stored local auth token.

## Notes

`.env.example` contains local development placeholder credentials only. Real credentials should be set in `.env` on the server.

## Verification

- `pytest -q`
- `npm run build`
- Protected endpoint returns `401` without credentials.
- `/auth/check` succeeds with configured credentials.
