# M5 - Weitere Buchmacher

## Status

Completed on 2026-04-24.

## Implemented

- Added DraftKings as a modular additional source adapter.
- Added DraftKings ENV controls:
  - `DRAFTKINGS_ENABLED`
  - `DRAFTKINGS_BASE_URL`
  - `DRAFTKINGS_REGION`
  - `DRAFTKINGS_TIMEOUT_SECONDS`
  - `DRAFTKINGS_REFRESH_INTERVAL_SECONDS`
- Added `POST /sources/draftkings/refresh`.
- Added parser coverage for DraftKings-style NBA player prop JSON.
- DraftKings failures are written to `refresh_runs` and `source_health` through the shared source runner.

## Source Audit

- DraftKings: lightweight JSON endpoint exists, but this server currently receives HTTP 403 from the public sportsbook edge. Adapter is present and disabled by default.
- FanDuel: no public developer API; public API access is undocumented and not stable enough for M5 without heavier reverse engineering.
- Bet365: skipped because stable player-prop scraping would likely require browser automation and high maintenance.
- Pinnacle: skipped because NBA player props are not reliably available without authenticated API access.
- Oddschecker: skipped for now because it would be HTML/browser-heavy for player props.
- ESPN Bet, BetMGM, Caesars: skipped for now because stable free player-prop access was not confirmed without undocumented/heavier scraping.

## Decision

M5 does not force unstable scrapers into production. Playzilla remains the mandatory source, DraftKings is prepared as the first additional adapter, and all other candidates are documented as skipped until a free, stable, lightweight access path is confirmed.
